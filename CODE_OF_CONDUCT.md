# Code of Conduct

## Our commitment

We are committed to providing a respectful, inclusive, and constructive environment for everyone who participates in the `kfc-procedure` project.

This project welcomes contributions from students, researchers, engineers, and open-source users with different levels of experience.

## Expected behavior

Participants are expected to:

- communicate respectfully and professionally,
- give constructive feedback,
- focus criticism on ideas and code rather than people,
- be patient with new contributors,
- acknowledge different experience levels and backgrounds,
- help keep discussions relevant to the project.

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, intimidation, or personal attacks,
- discriminatory language or behavior,
- insulting or dismissive comments toward contributors,
- publishing private information without permission,
- repeated off-topic disruption,
- any behavior that makes the project unsafe or unwelcoming.

## Scope

This code of conduct applies to all project spaces, including:

- GitHub issues,
- pull requests,
- discussions,
- wiki pages,
- documentation contributions,
- project-related communication channels.

## Enforcement

Project maintainers may remove comments, close issues, reject pull requests, or restrict participation when behavior violates this code of conduct.

If you experience or observe unacceptable behavior, contact the maintainer privately using the contact information in the repository metadata.

## Maintainer responsibility

Maintainers are responsible for clarifying standards, applying this code of conduct fairly, and protecting the health of the project community.
